## .gitignore

* В папке terraform/ используется файл .gitignore, который исключает из системы контроля версий следующие файлы и директории:

* .terraform/ — локальная директория с плагинами провайдеров и кэшем; создаётся автоматически при запуске terraform init.
* *.tfstate и *.tfstate.* — файлы состояния инфраструктуры, которые могут содержать конфиденциальные данные и изменяются при каждом применении конфигурации.
* crash.log и crash.*.log — логи аварийных завершений работы Terraform.
* *.tfvars, *.tfvars.json — файлы переменных, часто содержащие секреты (пароли, токены, ключи); их не рекомендуется хранить в репозитории.
* override.tf, override.tf.json, *_override.tf, *_override.tf.json — файлы локальных переопределений, используемые разработчиками на своих машинах и не предназначенные для общей версии.
* .terraform.tfstate.lock.info — временный файл блокировки, создаваемый при выполнении terraform apply.
* .terraformrc, terraform.rc — локальные конфигурационные файлы CLI, специфичные для пользователя.

Закомментированные правила (начинаются с #) показывают опциональные исключения, например, игнорирование файлов визуализации (*.dot) или планов (planout), но по умолчанию они не активны.
Символ * означает «любая последовательность символов». Например, *.tfstate.* игнорирует файлы вроде dev.tfstate.backup или prod.tfstate.123.
